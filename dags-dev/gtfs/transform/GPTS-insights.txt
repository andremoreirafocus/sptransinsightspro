agencies.txt -> não tem nenhuma informação relevante, pois só tem um registro com dados da SPTRANS. O agente_id está em routes.txt mas para todos o valor é “1”.

    routes.txt -> tem as linhas com seus códigos 
    route_id: primary_key 
    route_long_name: nome humano da rota. Ex: "Cem. Do Horto - Shop. Center Norte"
    route_type ?
    route_short_name =  route_id

trips.txt -> referencia uma Route_id como foreign key
	Trip_id: primary key
    route_id: foreign_key -< Se não for circular há dois registros com sentidos diferentes
    direction_id: sentido da trip. Ida: 0, Volta: 1 ->  info do routes.txt (route_long_name)
    trip_headsign: para onde está indo. Um para cada direction_id para o mesmo Route_id
        Ex: 	"1016-10","USD","1016-10-0","Shop. Center Norte","0","84176"
    "1016-10","USD","1016-10-1","Cem. Do Horto","1","84540"
        service_id: foreign key -> define os dias da semana em que a linha está ativa segundo o calendar.txt. 
    Importante: os horários dos ônibus são resultado da combinação do calendar.txt com o frequencies.txt que define em que horários do dia os ônibus rodam.
    shape_id: tem todas as posições do caminho (caminho GPS)
shapes.txt -> 
	shape_id + shape_pt_sequence: primary key (qual a shape e a posição no shape)
	shape_pt_lat e shape_pt_lon: posição
	shape_dist_traveled: distância percorrida desde o início do trajeto. 

stops.txt -> informações sobre as paradasS
	stop_id: primary key
	stop_name: human readable name
	stop_lat, stop_lon: posição

stop_times.txt -> tem a sequência de stops e horários previstos para cada stop de cada trip
	trip_id + stop_sequence: primary key
	stop_id: foreign key
	arrival_time: horario esperado para estar na primeira stop (horário de referência)
    departure_time: horario esperado para estar na stop considerando o horário da primeira stop
    stop_sequence: número sequencial do stop na trip

frequencies.txt -> determina os intervalos entre ônibus e que variam de acordo com diferentes horários do dia para cada linha
	trip_id: linha + sentido 
    start_time: início do período para o qual o intervalo é definido.
         junto com o trip_id compõe a primary key  
	end_time: início do período para o qual o intervalo é definido.
	headway_secs: intervalo em segundos

calendar.txt: define em que dias da semana a linha, se durante dias úiteis, sábado e domingo

fare_rules.txt: não tem relevância no escopo do projeto

fare_attributes.txt: não tem relevância no escopo do projeto
