# UQR (Unified Quality Result)

This document is the source of truth for the UQR produced by the transformlivedata pipeline. It defines the required inputs and the resulting output structure.

## Inputs

### Required Inputs to `build_uqr`

1. `execution_id` (string)
- Source: Generated in `transformlivedata-v7.py` at the start of the run.

2. `logical_date_utc` (string)
- Source: `logical_date_string` parameter passed into `load_transform_save_positions`.
- Format: ISO 8601 UTC string.

3. `source_file` (string)
- Source: Derived from `logical_date_utc` using the naming convention:
  - `posicoes_onibus-YYYYMMDDHHMM.json`

4. `transform_result` (dict)
- Source: Output of `transform_positions`.
- Required keys:
  - `positions` (DataFrame) with transformed valid candidate rows.
  - `invalid_positions` (DataFrame) with transform-stage invalid rows.
  - `metrics` (dict) with transformation metrics.
  - `issues` (dict) with transformation issues.

5. `valid_df` (DataFrame)
- Source: Output of `validate_expectations`.
- Meaning: Rows that passed GX expectations.

6. `invalid_df` (DataFrame or None)
- Source: Output of `validate_expectations`.
- Meaning: Rows rejected by GX expectations. Includes `invalid_reason` column.

7. `expectations_summary` (dict)
- Source: Output of `validate_expectations`.
- Required keys:
  - `total_checks` (int)
  - `checks_failed` (int)
  - `rows_failed` (int)
  - `failure_reasons` (list of `{rule, count}`)

8. `pass_threshold` (float)
- Source: Caller-provided policy value.

9. `warn_threshold` (float)
- Source: Caller-provided policy value.

### Input Responsibilities

- Raw validation is not part of UQR. If raw validation fails, the pipeline stops before UQR is built.
- Transformation failures are represented in `transform_result.invalid_positions` and are included in UQR counts.
- GX failures are represented in `invalid_df` and `expectations_summary`.

## Output

### UQR Schema (Current Implementation)

```json
{
  "execution_id": "uuid",
  "logical_date_utc": "2026-02-15T12:36:00Z",
  "source_file": "posicoes_onibus-202602150936.json",
  "row_counts": {
    "raw_records": 5251,
    "transformed_records": 5200,
    "accepted_records": 5150,
    "rejected_records": 50
  },
  "transformation_metrics": {
    "total_vehicles_processed": 5200,
    "valid_vehicles": 5150,
    "invalid_vehicles": 50,
    "expected_vehicles": 5251,
    "total_lines_processed": 120
  },
  "transformation_issues": {
    "invalid_trips": ["2104-10-0", "3063-1-1"],
    "invalid_vehicle_ids": [21300, 21303],
    "distance_calculation_errors": 3,
    "vehicle_count_discrepancies_per_line": 12
  },
  "expectations_summary": {
    "total_checks": 18,
    "checks_failed": 2,
    "rows_failed": 50,
    "failure_reasons": [
      {"rule": "expectation_type:expect_column_values_to_be_between|column:veiculo_lat", "count": 20},
      {"rule": "expectation_type:expect_column_values_to_be_between|column:veiculo_long", "count": 30}
    ]
  },
  "outcome": {
    "status": "WARN",
    "acceptance_rate": 0.9904,
    "policy_version": "v1"
  },
  "artifacts": {
    "quality_report_path": null,
    "quarantine_path": null
  }
}
```

### Field Definitions

- `row_counts.raw_records` is derived from `metrics.total_vehicles_processed` as agreed.
- `row_counts.transformed_records` is `len(transform_result.positions)`.
- `row_counts.accepted_records` is `len(valid_df)`.
- `row_counts.rejected_records` is `len(transform_result.invalid_positions) + len(invalid_df)`.

- `expectations_summary.failure_reasons` includes all failure categories.
- `outcome.status` is computed from acceptance thresholds and `checks_failed`:
  - PASS: `acceptance_rate >= pass_threshold` and `checks_failed == 0`
  - WARN: `acceptance_rate >= warn_threshold`
  - FAIL: otherwise

## Report Formatting

The UQR report is generated by `format_uqr_report` and logged once per run. Acceptance rate is formatted as a percentage with two decimals.
